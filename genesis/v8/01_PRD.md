# PRD — Genesis v8: Presentation Landing Framework

**Version**: 8.0  
**Date**: 2026-05-14  
**Status**: Draft  
**Author**: Antigravity / Genesis Agent  
**Product**: Expoint ADV — Единая презентационная система страниц

---

## 1. Problem Statement

Платформа Expoint ADV содержит **20+ section-компонентов**, готовую дизайн-систему (Verge 2024), коммерческий слой (v7), и SEO-инфраструктуру, но:

- **Нет единого фреймворка страниц** — каждая страница собирается вручную (ad-hoc импорт компонентов).
- **Отсутствуют ключевые страницы** — О компании, Контакты, Портфолио, Архив кейсов, полноценная Академия.
- **Навигация фрагментирована** — Header содержит только anchor-ссылки на главную, нет полноценной маршрутной навигации.
- **Нет переиспользуемого шаблона** — создание новой страницы требует ручной сборки layout + секций + SEO.

**Цель**: Создать **Presentation Landing Framework** — единый data-driven фреймворк, позволяющий декларативно описывать и рендерить любую страницу сайта из готовых секций, и на его основе создать все необходимые наполненные страницы.

## 3. User Stories

### US-001 — Единый Page Shell

> Как разработчик, я хочу иметь единый layout-компонент для всех страниц, чтобы не дублировать Header/Footer/модальные окна.

**Acceptance Criteria**:
- [REQ-101] `PageShell` компонент принимает `children`, `headerVariant`, `showFooter`, `showAssistant`.
- [REQ-102] Включает Header (с вариативностью), Footer, ConsultationModal, CookieBanner, AssistantWidget.
- [REQ-103] Поддерживает `breadcrumbs` prop для отображения навигационных хлебных крошек.
- [REQ-104] Используется на **всех** страницах вместо ручной сборки.

### US-002 — Section Registry & Page Renderer

> Как разработчик, я хочу декларативно описать страницу как массив секций с параметрами и получить готовый рендер.

**Acceptance Criteria**:
- [REQ-110] `SectionRegistry` — объект, маппящий `sectionType` → React-компонент.
- [REQ-111] `PageRenderer` — компонент, принимающий `sections: SectionConfig[]` и рендерящий их последовательно.
- [REQ-112] Каждый `SectionConfig` содержит: `type`, `id`, `props`, `condition?`.
- [REQ-113] Поддержка `condition` для условного рендеринга (например, на мобильных).

### US-003 — Navigation System

> Как пользователь сайта, я хочу видеть полноценное навигационное меню с доступом ко всем разделам.

**Acceptance Criteria**:
- [REQ-120] Header содержит навигационные ссылки на все основные маршруты: Услуги, Цены, Каталог, Кейсы, Академия, О нас, Контакты.
- [REQ-121] Активный маршрут выделяется визуально.
- [REQ-122] Мобильное меню содержит все маршруты + CTA.
- [REQ-123] Breadcrumbs отображаются на всех внутренних страницах.

### US-004 — Страница «О компании» (`/about`)

> Как потенциальный клиент, я хочу узнать о компании: историю, производство, команду, преимущества — чтобы убедиться в надёжности подрядчика.

**Acceptance Criteria**:
- [REQ-130] Маршрут `/about` в route group `(marketing)`.
- [REQ-131] Hero-секция с заголовком, описанием компании и фоновым изображением производства.
- [REQ-132] Секция «Цифры и факты» — статистика (лет на рынке, проектов, клиентов, м² продукции).
- [REQ-133] Секция «Наше производство» — фотогалерея / видео цеха.
- [REQ-134] Секция «Миссия и ценности» — 3-4 карточки ценностей.
- [REQ-135] Секция «Команда» — фотокарточки ключевых сотрудников.
- [REQ-136] CTA-секция «Начать проект».
- [REQ-137] SEO: title, description, schema.org `Organization`.

### US-005 — Страница «Контакты» (`/contacts`)

> Как клиент, я хочу быстро найти адрес, телефон, email и карту проезда, чтобы связаться или приехать.

**Acceptance Criteria**:
- [REQ-140] Маршрут `/contacts` в route group `(marketing)`.
- [REQ-141] Hero-секция с заголовком «Связаться с нами».
- [REQ-142] Блок с контактной информацией: адрес, телефон, email, Telegram, WhatsApp, график работы.
- [REQ-143] Карта (MapSection) в широкоформатном варианте.
- [REQ-144] Встроенная форма обратной связи (переиспользуется из ConsultationModal).
- [REQ-145] SEO: schema.org `LocalBusiness`.

### US-006 — Страница «Портфолио» (`/portfolio`)

> Как клиент, я хочу видеть галерею реализованных проектов, чтобы оценить уровень работ.

**Acceptance Criteria**:
- [REQ-150] Маршрут `/portfolio` в route group `(marketing)`.
- [REQ-151] Grid-галерея с фильтрацией по сегменту (retail, horeca, corporate).
- [REQ-152] Карточка проекта: изображение, название, сегмент, бюджет, сроки.
- [REQ-153] Модальный просмотр деталей по клику.
- [REQ-154] CTA-секция «Обсудить ваш проект».

### US-007 — Страница «Архив кейсов» (`/cases`)

> Как B2B-клиент, я хочу изучить кейсы с метриками, чтобы оценить ROI.

**Acceptance Criteria**:
- [REQ-160] Маршрут `/cases` уже существует — нужно наполнить полноценным контентом.
- [REQ-161] Список кейсов с фильтрацией по сегменту.
- [REQ-162] Каждый кейс: клиент, задача, решение, метрики, бюджет, сроки.
- [REQ-163] CTA «Заказать аналогичное решение» → корзина или калькулятор.

### US-008 — Страница «Академия» (`/academy`)

> Как клиент или дизайнер, я хочу получать полезный контент: статьи, гайды, чек-листы по наружной рекламе.

**Acceptance Criteria**:
- [REQ-170] Маршрут `/academy` с индексной страницей.
- [REQ-171] Grid карточек статей из MDX-контента.
- [REQ-172] Фильтрация по тегам (материалы, законодательство, кейсы, технологии).
- [REQ-173] SEO: schema.org `Article` для каждой статьи.

### US-009 — Единые Shared-секции

> Как разработчик, я хочу иметь набор готовых переиспользуемых секций для быстрой сборки.

**Acceptance Criteria**:
- [REQ-180] `HeroGeneric` — универсальный Hero с props: title, subtitle, description, bgImage, cta.
- [REQ-181] `StatsSection` — секция статистики с анимированными числами.
- [REQ-182] `CTASection` — призыв к действию с заголовком, описанием и кнопкой.
- [REQ-183] `TestimonialsSection` — отзывы клиентов в карусели.
- [REQ-184] `TeamSection` — карточки команды.
- [REQ-185] `ValuesSection` — карточки ценностей/преимуществ.
- [REQ-186] `ContactInfoSection` — контактная информация + форма.
- [REQ-187] `GalleryGrid` — адаптивная фото-галерея с lightbox.
- [REQ-188] `BreadcrumbsBar` — навигационные хлебные крошки.

## 4. Non-Functional Requirements

- [NFR-101] SEO: Все страницы имеют `generateMetadata`, `og:*`, и schema.org разметку.
- [NFR-102] Performance: LCP < 2.5s для всех статических страниц (SSG).
- [NFR-103] Accessibility: ARIA roles на интерактивных элементах, alt на изображениях.
- [NFR-104] Motion: Все секции используют Framer Motion `whileInView` для scroll-анимаций.
- [NFR-105] Responsive: Все страницы корректно отображаются от 320px до 2560px.
- [NFR-106] i18n: Все текстовые элементы поддерживают мультиязычность через `LocalizedText`.
- [NFR-107] Design System: Все компоненты следуют **новой светлой дизайн-системе** (Light-first). Дизайн-токены обновлены для светлого фона с сохранением бренд-акцентов.
- [NFR-108] Theme: Основная тема — **светлая (Light)**. Dark mode доступен как альтернатива через ThemeToggle.
- [NFR-109] Palette: Новая цветовая палитра адаптирована для светлого фона с высоким контрастом текста и акцентов.
