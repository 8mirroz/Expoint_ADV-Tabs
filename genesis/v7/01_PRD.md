# PRD — Genesis v7: Commerce Layer (Prices · Calculator · Catalog · Cart)

**Version**: 7.0  
**Date**: 2026-05-13  
**Status**: Draft  
**Author**: Antigravity / Genesis Agent  
**Product**: Expoint ADV — Производство наружной рекламы

---

## 1. Problem Statement

Платформа Expoint ADV имеет развитую маркетинговую архитектуру (лендинги услуг, анимации, SEO), но лишена коммерческого слоя:  
- Нет публичной страницы с ценами — клиент не может самостоятельно оценить бюджет.  
- Калькулятор существует как изолированный компонент без маршрута и страницы.  
- Каталог готовых решений (`PRODUCT_PACKS`) не имеет UI.  
- Нет корзины — все конверсии происходят только через лид-форму.  

**Цель**: Создать полный коммерческий слой платформы, закрывающий воронку от информирования до оформления заказа.

---

## 2. Goals & Success Metrics

| Цель | Метрика |
|---|---|
| Клиент видит ориентировочную стоимость | Bounce rate на `/prices` < 60% |
| Конфигуратор завершается заявкой | Conversion: step 4 → submit > 15% |
| Корзина снижает барьер входа | Средний чек в корзине > 1 позиции |
| Каталог ускоряет принятие решения | CTR "Добавить в корзину" > 8% |

## 3. User Stories

### US-001 — Страница цен (`/prices`)
> Как потенциальный клиент (B2B, HoReCa), я хочу видеть ценовые диапазоны по каждой услуге, чтобы понять, подхожу ли я по бюджету до звонка менеджеру.

**Acceptance Criteria**:
- [REQ-001] Страница `/prices` отображает тарифные сетки по всем `SERVICES` с `basePrice` и `priceUnit`.
- [REQ-002] Для каждой услуги есть CTA-кнопка "Рассчитать точнее" → переход на `/calculator?type={serviceId}`.
- [REQ-003] Отображаются `PRODUCT_PACKS` с ценой входа и составом.
- [REQ-004] Есть раздел "Что влияет на цену" (факторы: высота, материал, срочность, монтаж).
- [REQ-005] Страница статически рендерится (SSR/SSG), SEO-оптимизирована.
- [REQ-006] На мобильных — аккордеон по услугам.

### US-002 — Страница калькулятора (`/calculator`)
> Как клиент, желающий получить точный расчёт, я хочу настроить параметры вывески и получить диапазон стоимости с 3D-превью, чтобы добавить конфигурацию в заявку.

**Acceptance Criteria**:
- [REQ-010] Маршрут `/calculator` с `layout.tsx` и `page.tsx` в `(calculator)` route group.
- [REQ-011] `CalculatorContainer` встроен в страницу с `Suspense`/`dynamic`.
- [REQ-012] Параметр `?type=` предзаполняет `productType` в store при открытии.
- [REQ-013] На шаге "Спецификация" есть кнопка "Добавить в заявку" → `CartStore.addCustomItem()`.
- [REQ-014] После добавления — toast-уведомление + ссылка на `/cart`.
- [REQ-015] Страница SEO-оптимизирована (`generateMetadata`).

### US-003 — Каталог готовых решений (`/catalog`)
> Как клиент, который не хочет разбираться в деталях, я хочу выбрать готовый пакет "под ключ" и сразу добавить его в заявку.

**Acceptance Criteria**:
- [REQ-020] Маршрут `/catalog` в `(catalog)` route group.
- [REQ-021] Отображаются все `PRODUCT_PACKS` в виде карточек с визуальным выделением `isPopular`.
- [REQ-022] Карточка содержит: название, описание, целевой сегмент, цену от, список фич.
- [REQ-023] Кнопка "Добавить в заявку" → `CartStore.addPackItem(pack)`.
- [REQ-024] Фильтрация по сегменту бизнеса (horeca, retail, corporate, franchise).
- [REQ-025] Мини-конфигуратор внутри карточки: выбор подсветки (стандарт/премиум) с пересчётом стартовой цены.
- [REQ-026] Каждая карточка ссылается на полный лендинг услуги `/services/{serviceId}`.

### US-004 — Корзина (`/cart`)
> Как клиент, добавивший позиции, я хочу просмотреть итоговый список, при необходимости удалить/изменить позиции, и отправить заявку менеджеру.

**Acceptance Criteria**:
- [REQ-030] `CartStore` (Zustand) с персистентностью в `localStorage`.
- [REQ-031] Типы позиций: `pack` (из каталога) и `custom` (из калькулятора).
- [REQ-032] Маршрут `/cart` с полным UI: список позиций, суммарная стоимость, форма контакта.
- [REQ-033] Форма: имя, телефон/email, комментарий. Валидация через Zod + react-hook-form.
- [REQ-034] Submit → `POST /api/leads` с `cartSnapshot` в теле запроса.
- [REQ-035] После успешной отправки — страница "Спасибо" + очистка корзины.
- [REQ-036] Если корзина пуста — редирект на `/catalog`.
- [REQ-037] Счётчик позиций корзины отображается в `Header`.
- [REQ-038] Корзина сохраняется между перезагрузками (localStorage).

### US-005 — Интеграция цен в лендинги услуг
> Как клиент на странице `/services/volumetric-letters`, я хочу видеть ориентировочную цену прямо на лендинге.

**Acceptance Criteria**:
- [REQ-040] На каждом `[slug]/page.tsx` есть блок с `basePrice`/`priceUnit` из `SERVICES`.
- [REQ-041] Есть CTA "Рассчитать стоимость" → `/calculator?type={slug}`.
- [REQ-042] Есть CTA "Смотреть готовые пакеты" → `/catalog?segment={segment[0]}`.

## 4. Non-Functional Requirements

- [NFR-001] SEO: все страницы имеют `title`, `description`, `og:*`, schema.org `Service` или `Product`.
- [NFR-002] Performance: LCP `/prices` < 2.5s (SSG-рендеринг).
- [NFR-003] Аналитика: все действия (добавление в корзину, submit, шаги калькулятора) трекируются через `useAnalytics`.
- [NFR-004] Безопасность: `/api/leads` защищён CSRF-токеном и rate limiting (middleware).
- [NFR-005] Accessibility: ARIA-роли на интерактивных элементах корзины и форм.

---

## 5. Out of Scope (v7)

- Онлайн-оплата (будет в v8).
- Личный кабинет / история заказов.
- Автоматическая выдача КП (PDF-генерация).
- Административная панель управления ценами.
