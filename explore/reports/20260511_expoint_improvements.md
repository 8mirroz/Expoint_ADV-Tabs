# Exploration Report: Упущенные возможности и точки роста Expoint ADV

**Date**: 2026-05-11
**Explorer**: AI Explorer (Antigravity)

---

## 1. Problem & Scope

**Core Question**: Что упущено в текущей реализации проекта Expoint ADV и как можно его улучшить для достижения уровня Tier-1 Enterprise?

**Exploration Scope**:
- Included: UX/UI (AntiQ v3), Architecture (React/Vite/Zustand), Conversion Layer, AI Integration, DevOps & SEO.
- Excluded: Изменение бизнес-модели, смена движка на другой фреймворк (без веских причин).

---

## 2. Key Insights

> Анализ текущей базы кода и архитектуры выявил 4 ключевые зоны для улучшения: от замены 2D превью на 3D до решения проблемы SEO, которая критична для SPA на Vite.

1. **SPA vs SEO Limitation**: Текущий стек (Vite SPA) не индексируется поисковиками должным образом, что критично для B2B-лидогенерации.
2. **2D to 3D WOW-factor**: Текущее SVG-превью работает хорошо, но для премиум-сегмента интеграция Three.js/React-Three-Fiber поднимет конверсию за счет реалистичного 3D-рендера вывески.
3. **Отсутствие Data Pipeline**: Калькулятор считает цену, но не передает данные ни в CRM (AmoCRM/Bitrix), ни в систему аналитики (Яндекс.Метрика).
4. **Mocked AI Backend**: ИИ-консультант работает на замоканных таймаутах. Необходим реальный API-прокси (Vercel AI SDK + LLM).

---

## 3. Detailed Findings

### 3.1 Архитектура и SEO (SPA Limitation)
**Exploration Method**: 🧠 Diverge
**Findings**:
- **Проблема**: Vite собирает проект как Single Page Application (SPA). Весь контент рендерится на клиенте. Поисковые роботы (Яндекс, Google) будут видеть пустой HTML. Для бизнеса наружной рекламы органический трафик — один из главных каналов.
- **Решение**: Миграция на **Next.js (App Router)** или настройка **Vite Plugin SSR / SSG (Vike)**. Учитывая текущий чистый React-код, перенос на Next.js займет минимум времени, но даст серверный рендеринг (SSR), мета-теги для каждой страницы и генерацию sitemap.

### 3.2 Вау-эффект Конфигуратора (UX/UI)
**Exploration Method**: 🧠 Diverge / 🔍 Mixed
**Findings**:
- **Текущее состояние**: Калькулятор использует SVG с CSS-фильтрами (neon glow). Это функционально, но не выглядит "дорого".
- **Инновация**: Использование `react-three-fiber` для создания 3D-сцены. Пользователь сможет вращать вывеску, видеть объемные буквы (ExtrudeGeometry), игру света на стене (PointLight) и материалы (металл, акрил). Это полностью соответствует гайдлайнам AntiQ v3 (Luxury/Industrial).

### 3.3 Конверсионный пайплайн и CRM
**Exploration Method**: 🧠 Inward
**Findings**:
- Стейт конфигуратора (текст, размер, цвет, монтаж, цена) хранится в Zustand, но кнопка "Оформить заказ" не привязана к backend-обработчику.
- Необходимо создать API-роут, который:
  1. Формирует PDF-коммерческое предложение или заявку.
  2. Отправляет данные через Webhook в Telegram / AmoCRM / Bitrix24.
  3. Отправляет Event в системы аналитики (Conversion Goal).

### 3.4 Реализация AI-ассистента
**Exploration Method**: 🔍 Search
**Findings**:
- Сейчас виджет имитирует работу через `setTimeout`.
- Для Production требуется бэкенд на Node.js (или Next.js Route Handlers) с `@ai-sdk/google` (Gemini) или OpenAI.
- Контекст из `useAssistantContext` должен отправляться в массиве сообщений как `system prompt`, чтобы модель не галлюцинировала, а продавала именно ту вывеску, которую собирает клиент.

## 5. Action Recommendations

| Priority | Recommendation | Reason |
|:--------:|----------------|--------|
| **P0** | **CRM & Analytics Integration** | Без сохранения лидов калькулятор не приносит денег. Нужно сделать POST-запрос при клике на "Оформить заказ". |
| **P1** | **Миграция на Next.js (или SSR)** | Если сайт пойдет в релиз для SEO-продвижения, SPA убьет органический трафик. |
| **P1** | **Реальный AI Backend** | Подключение Gemini API для работы ИИ-ассистента в проде. |
| **P2** | **3D Visualizer (Three.js)** | Важный имиджевый шаг, но требует времени на создание 3D-шрифтов и настройку освещения. |
| **P3** | **Headless CMS** | Замена статических файлов `portfolio.ts` на API-вызовы, если контент-менеджерам нужен интерфейс. |

## 6. Limitations & To-Explore

- Уточнить у бизнеса: как именно они хотят получать заявки (Telegram, Email, CRM)?
- Согласовать необходимость SEO: если трафик только из Директа/Таргета, можно оставить Vite SPA и просто добавить аналитику. Но для долгосрочной стратегии SEO необходимо.
- Проверить производительность 3D-рендера на мобильных устройствах перед внедрением Three.js.
