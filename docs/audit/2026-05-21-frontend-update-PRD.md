# PRD: Обновление фронтенда Expoint ADV

**Версия:** 1.0
**Дата:** 21 мая 2026
**Автор:** Frontend Audit Team
**Основание:** Комплексный аудит фронтенда от 21.05.2026

---

## Problem Statement

Expoint ADV (БУКВА СВЕТ) — B2B digital-платформа для производства наружной рекламы, находящаяся на стадии 68% готовности к production-запуску. Аудит выявил системные проблемы в 5 областях:

1. **Производительность (62/100):** Блокирующие ресурсы, неоптимизированные изображения, избыточный клиентский JS
2. **Доступность (58/100):** Отсутствие ключевых WCAG landmarks, проблемы с контрастностью и семантикой
3. **SEO (55/100):** sitemap с 1 URL из ~20, отсутствие структурных данных Organization/WebSite
4. **UX (78/100):** Нет skeleton loaders, неполные empty states
5. **Код (82/100):** Лишние зависимости (+50KB), дублирование i18n, параллельные button systems

Без устранения этих проблем запуск 27 мая — 3 июня 2026 несёт риски низкой индексации, медленной загрузки и жалоб на доступность.

## Solution

Трёхфазное обновление фронтенда с приоритетом: **SEO → Performance → Accessibility → UX → Code Quality**. Каждая фаза имеет чёткие критерии приёмки и может быть развёрнута независимо.

---

## Цели и задачи обновления

### Бизнес-цели
1. Обеспечить полную индексацию всех ~20 страниц в Google/Yandex к запуску
2. Улучшить Core Web Vitals до зелёной зоны (LCP <2.5s, CLS <0.1, INP <200ms)
3. Соответствие WCAG 2.1 AA для B2B-клиентов с ограниченными возможностями
4. Повысить конверсию заявок на 15-20% через улучшение perceived performance

### Пользовательские задачи
1. Пользователь видит контент быстрее (FCP <1.5s)
2. Пользователь может навигировать с клавиатуры
3. Поисковые системы находят все страницы и услуги
4. B2B-клиенты получают профессиональное впечатление от скорости и качества

## Область применения (Scope)

### В scope:
- Все страницы маркетинговой группы `(marketing)/*`
- Layout (`layout.tsx`, `globals.css`, `tokens.css`)
- Компоненты навигации (Header, Footer, BreadcrumbsBar)
- SEO-инфраструктура (sitemap, robots, schema)
- Оптимизация изображений (20 файлов с `<img>`)
- Очистка зависимостей (package.json)
- Accessibility landmarks и семантика

### Вне scope:
- Backend API routes (без изменений)
- Database schema (без изменений)
- 3D-визуализация (не используется на production-страницах)
- AI-консультант (beta-режим, отдельный scope)
- Новый функционал калькулятора
- Мобильное приложение

## Функциональные требования

### FR-1: SEO-инфраструктура
- **FR-1.1:** sitemap.ts содержит все статические маршруты (~20 страниц) с корректными `changeFrequency` и `priority`
- **FR-1.2:** robots.ts содержит `disallow: ['/api/', '/admin/', '/private/']`
- **FR-1.3:** Добавлена schema Organization для главной страницы
- **FR-1.4:** Добавлена schema WebSite с SearchAction
- **FR-1.5:** og-image.jpg присутствует в `/public/` (1200x630)
- **FR-1.6:** Каждая страница имеет ровно 1 элемент `<h1>`

### FR-2: Оптимизация изображений
- **FR-2.1:** Все `<img>` заменены на `next/image`
- **FR-2.2:** Above-the-fold изображения имеют `priority={true}`
- **FR-2.3:** Ниже fold — `loading="lazy"` (по умолчанию в next/image)
- **FR-2.4:** Все изображения имеют `alt` атрибут

### FR-3: Оптимизация ресурсов
- **FR-3.1:** Material Symbols загружается через `next/font` или `<link rel="preload">`
- **FR-3.2:** Удалены неиспользуемые зависимости: `@nestjs/config`, `express`, `@splinetool/react-spline`, `three-stdlib`
- **FR-3.3:** Объединены `framer-motion` и `motion` в один пакет (`motion`)

### FR-4: Accessibility
- **FR-4.1:** layout.tsx содержит `<main id="main-content">`
- **FR-4.2:** Добавлена skip-navigation ссылка
- **FR-4.3:** Все формы имеют `<label>` или `aria-label`
- **FR-4.4:** Минимальная контрастность текста: AA 4.5:1

### FR-5: UX-улучшения
- **FR-5.1:** Skeleton loaders для Cards, Gallery, Calculator секций
- **FR-5.2:** Exit-анимация ConsultationModal

## User Stories

### Производительность
1. As a **B2B-клиент** на медленном соединении, I want **страница загрузилась за <3s**, so that **я не ушёл к конкуренту**
2. As a **пользователь мобильного**, I want **изображения загружались по мере прокрутки**, so that **экономился трафик**
3. As a **посетитель главной**, I want **шрифты не мигали при загрузке**, so that **первое впечатление было премиальным**

### SEO
4. As a **менеджер по маркетингу**, I want **все 20 страниц индексировались в Google**, so that **органический трафик рос**
5. As a **поисковый бот**, I want **корректный sitemap и robots.txt**, so that **эффективно обходить сайт**
6. As a **пользователь Google**, I want **видеть rich snippets с рейтингом и FAQ**, so that **доверять компании до клика**

### Доступность
7. As a **пользователь скринридера**, I want **навигировать по landmarks (header, main, footer)**, so that **быстро перемещаться по странице**
8. As a **пользователь с клавиатуры**, I want **skip-navigation ссылку**, so that **пропустить шапку и сразу читать контент**
9. As a **пользователь с нарушением зрения**, I want **достаточный контраст текста**, so that **читать без напряжения**

### UX
10. As a **пользователь**, I want **видеть skeleton-загрузку**, so that **понимать что контент появится**
11. As a **пользователь калькулятора**, I want **плавное закрытие модальных окон**, so that **интерфейс ощущался отполированным**

### Код
12. As a **разработчик**, I want **видеть test coverage report**, so that **знать какие модули покрыты**
13. As a **разработчик**, I want **не иметь лишних зависимостей**, so that **бандл был минимальным**

## Implementation Decisions

### Modules to modify:
1. **SEO Module** — sitemap.ts, robots.ts, schema.ts (3 файла)
2. **Layout Module** — layout.tsx, PageShell.tsx (2 файла)
3. **Image Module** — 20 файлов с `<img>` → `next/image`
4. **CSS Module** — globals.css, tokens.css (2 файла)
5. **Dependencies** — package.json (1 файл)
6. **Form Accessibility** — Hero.tsx, LeadForm.tsx, ConsultationModal.tsx (~5 файлов)

### Architectural decisions:
- Не создаём новых абстракций
- Каждое изменение — минимальное, изолированное, тестируемое
- Все изменения совместимы с текущей dark-theme forced mode
- i18n centralization — future phase, не блокирует запуск

### Testing decisions:
- Unit tests для schema generators
- Visual regression: ручная проверка после каждой фазы
- Build verification после каждого PR
