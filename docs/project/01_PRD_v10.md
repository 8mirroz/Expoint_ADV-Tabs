# Expoint ADV — PRD v10: B2B Inner Pages Architecture

## Executive Summary

Expoint ADV v10 переходит от информационных внутренних страниц к **конверсионной B2B-архитектуре**. Каждая внутренняя страница становится самостоятельным инструментом продаж, построенным на данных из NotebookLM knowledge base (7 coverage файлов), дизайн-системе Vercel Geist Light Theme и Presentation Framework Core (PFC).

## Core Objectives v10

### 1. Pricing Trust-Machine

Страница `/prices` трансформируется из статичного прайс-листа в **конверсионный инструмент**, отвечающий на 5 вопросов B2B-клиента:

1. **Сколько стоит?** → Pricing Preview Panel + Service Rates
2. **Почему именно столько?** → Pricing Explainer (5 драйверов)
3. **От чего зависит цена?** → Scenario Packages (Start/Business/Premium/Network)
4. **Как не переплатить и не получить плохой результат?** → Compliance/Risk блок
5. **Что делать дальше?** → Калькулятор + WhatsApp + CTA

### 2. Premium Visual Parity

Все внутренние страницы приводятся к визуальному уровню главной:
- Engineering green accent → `--geist-ink` as primary
- TechnicalGrid + MeshBackground on heroes
- Geist Display typography scale (fluid)
- Glassmorphism cards for premium feel
- Subtle motion (IntersectionObserver reveals)
- Blueprint grid pattern

### 3. Knowledge-Driven Content

Контент формируется из проверенных knowledge-файлов:

| Файл | Тип данных | Применение |
|---|---|---|
| t1 | Конкурентный анализ neonovye-vyveski.ru | Hero-шаблоны, benefit cards, FAQ, case structure |
| t2 | B2B Pricing Strategy + Ultra Prompt | Page architecture, pricing funnel, SEO-запросы |
| t3_neon | Гибкий неон deep-dive | ServiceRates для flex-neon, technology comparison |

| t6_navigation | Навигация и стелы | ServiceRates для pylon-signs, segment packages |
| t7_setup | Монтаж и демонтаж | PricingCompliance, risk mitigation, installation costs |

### 4. SEO/AEO Readiness

Каждая страница оптимизирована для:

## Technical Architecture v10

### Stack
- **Framework**: Next.js 15 (App Router)
- **UI**: React 19, motion/react, lucide-react
- **Styling**: TailwindCSS v4, tokens.css (Vercel Geist)
- **Type System**: TypeScript strict
- **Rendering**: PFC (PageBlueprint → SectionRegistry → lazy components)

### Data Flow
```
docs/knowledge/coverage/t1-t7
        ↓
src/data/pages/prices.ts (PageBlueprint)
        ↓
src/data/services.ts (Service[])
        ↓
SectionRegistry → Components
        ↓
/prices page render
```

### Component Architecture
```
PageShell (withMesh, headerVariant)
  └── PageRenderer
       ├── HeroGeneric (+ pricingPreview panel)
       ├── PricingExplainer (numbered drivers)
       ├── ServiceRates (complexity indicators)
       ├── Pricing (4 scenario packages)
       ├── PricingCompliance (severity markers)
       ├── StatsSection
       ├── FAQ
       └── CTASection
```
